COMPRAS - STATUS
- A tabela oficial de status dos pedidos de compra e cp_compras_status.
- O pedido grava o codigo do status em cp_compras.status_id.
- O sistema interno deve apresentar sempre cp_compras_status.descricao_compras.
- O campo cp_compras_status.descricao_portal e reservado para o portal do fornecedor.
- Nao montar status a partir da Localizacao do pedido.
- Localizacao deve aparecer somente em campo/coluna propria, com valores como KidStok ou Fornecedor.
- Nao apresentar status derivados como "Aberto Aguardando KidStok" ou "Aberto Aguardando Fornecedor" se estes textos nao existirem em cp_compras_status.descricao_compras.

Status oficiais de cp_compras_status:
- 0 - Aberto
- 1 - Aprovado Aguardando Foto Fornecedor
- 2 - Aprovado
- 3 - Recusado

Onde houver apresentacao de status no sistema interno:
- cp_compras_lista
- dashboard
- cp_compras_form
- PDF de pedido de compra

deve ser usado o valor de cp_compras_status.descricao_compras.

COMPRAS - RATEIO
- O rateio percentual de compras deve ser informado por item + cor.
- A tabela cp_compras_itens_rateios relaciona compras_itens_id + cor e grava o percentual da cor no item.
- Se o item nao tiver percentuais de rateio, ao informar a quantidade do tamanho o sistema divide igualmente entre as cores ativas.
- Se houver qualquer percentual informado no item, a soma dos percentuais deve fechar 100%.
- Todos os tamanhos ativos do item devem ter a mesma quantidade e o mesmo conjunto de cores para permitir rateio percentual.
- A quantidade total fica no tamanho, no campo cp_compras_itens_tamanhos.qtde_total.
- As quantidades das cores de cada tamanho devem somar a qtde_total do tamanho.
- A data de entrega do tamanho e opcional. Se nao for informada no campo do tamanho, gravar NULL em cp_compras_itens_tamanhos.entrega e cp_compras_itens_tamanhos.entrega_anterior, sem assumir a data do item nem a data do pedido.
- A quantidade da cor pode ser ajustada manualmente, mas a soma das cores nao pode ultrapassar a quantidade total do tamanho.
- Item, tamanho ou cor inativo nao deve validar quantidade/rateio; item e tamanho inativos devem zerar os totais de quantidade.
- No accordion fechado do item deve haver botao para informar o rateio percentual das cores.
- No accordion do tamanho deve ser permitida a quantidade total do tamanho.

COMPRAS - INTERACOES DO FORMULARIO
- Ao salvar pedido, o botao Salvar deve ficar desabilitado com indicador de processamento ate terminar a requisicao.
- Pressionar Enter em inputs do formulario de compras nao deve salvar o pedido automaticamente.
- Ao focar qualquer input do formulario de compras ou do modal de rateio, o conteudo deve ser selecionado automaticamente.
- Excluir item, tamanho, cor ou foto deve solicitar confirmacao.

COMPRAS - FOTOS

- Fotos do fornecedor usam cp_compras_fotos no banco allop_devel_fotos.

- Quando o pedido estiver somente leitura por localizacao/status, fotos ficam apenas para visualizacao.

- Inputs, selects, input-group-text e Select2 devem usar altura compacta de 38px.

- Foco de campos e Select2 deve usar laranja #ff4500 com sombra leve.

Padrao do header da tela:
- Usar render_header('Titulo', [...acoes...]) quando houver botao Voltar.
